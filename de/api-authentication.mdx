## Authentifizierungsmethoden

AetherFlow unterstuetzt mehrere Authentifizierungsmethoden, um einen sicheren API-Zugriff zu gewaehrleisten. Waehlen Sie die geeignete Methode basierend auf Ihrem Anwendungsfall und Ihren Integrationsanforderungen.

<Callout kind="info">
  Alle API-Anfragen muessen ueber HTTPS gestellt werden. HTTP-Anfragen werden abgelehnt.
</Callout>

## JWT-Token-Authentifizierung

JWT (JSON Web Tokens) ermoeglicht eine zustandslose Authentifizierung fuer den programmgesteuerten Zugriff. Generieren Sie Token in Ihren Kontoeinstellungen fuer sichere API-Interaktionen.

<Steps>
  <Step title="JWT-Token generieren" icon="key">
    Navigieren Sie zu Einstellungen > API-Schluessel in Ihrem Dashboard. Erstellen Sie ein neues Token mit geeigneten Bereichen.
  </Step>
  <Step title="In Anfragen einfuegen" icon="send">
    Fuegen Sie den Authorization-Header zu allen API-Aufrufen hinzu: `Authorization: Bearer YOUR_JWT_TOKEN`
  </Step>
  <Step title="Token-Verwaltung" icon="settings">
    Rotieren Sie Token regelmaessig und widerrufen Sie kompromittierte sofort.
  </Step>
</Steps>

```javascript
// Beispiel-API-Anfrage mit JWT
const response = await fetch('https://api.aetherflow.com/v2/workflows', {
  method: 'GET',
  headers: {
    'Authorization': `Bearer ${process.env.AETHERFLOW_JWT_TOKEN}`,
    'Content-Type': 'application/json'
  }
});
```

## API-Schluessel-Authentifizierung

Legacy-API-Schluessel werden zur Rueckwaertskompatibilitaet noch unterstuetzt. Wir empfehlen jedoch, fuer verbesserte Sicherheit auf JWT-Token zu migrieren.

<Tabs>
  <Tab title="API-Schluessel erstellen" icon="plus">
    Greifen Sie auf Ihren API-Schluessel-Bereich zu und generieren Sie einen neuen Schluessel mit spezifischen Berechtigungen.

    <CodeGroup tabs="cURL,Python">
      ```bash
      curl -X GET "https://api.aetherflow.com/v1/workflows" \
        -H "X-API-Key: YOUR_API_KEY" \
        -H "Content-Type: application/json"
      ```
      ```python
      import requests

      headers = {
          'X-API-Key': 'YOUR_API_KEY',
          'Content-Type': 'application/json'
      }

      response = requests.get('https://api.aetherflow.com/v1/workflows', headers=headers)
      ```
    </CodeGroup>
  </Tab>

  <Tab title="Schluessel-Bereiche" icon="shield">
    Begrenzen Sie API-Schluessel-Berechtigungen auf bestimmte Vorgaenge:
    - `read:workflows` - Workflow-Daten anzeigen
    - `write:workflows` - Workflows erstellen und aendern
    - `execute:workflows` - Workflows programmgesteuert ausfuehren
  </Tab>
</Tabs>

## OAuth-2.0-Integration

Fuer Drittanbieter-Anwendungen implementieren Sie OAuth-2.0-Flows, um sicher auf Benutzerdaten zuzugreifen.

<Expandable title="OAuth-Flow-Implementierung">
  ```javascript
  // Erstellung der Autorisierungs-URL
  const authUrl = `https://api.aetherflow.com/oauth/authorize?` +
    `client_id=${CLIENT_ID}&` +
    `redirect_uri=${encodeURIComponent(REDIRECT_URI)}&` +
    `response_type=code&` +
    `scope=workflows:read workflows:write`;

  // Code gegen Token austauschen
  const tokenResponse = await fetch('https://api.aetherflow.com/oauth/token', {
    method: 'POST',
    headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
    body: `grant_type=authorization_code&code=${authCode}&client_secret=${CLIENT_SECRET}`
  });
  ```
</Expandable>

## Sicherheits-Best-Practices

Implementieren Sie diese Sicherheitsmassnahmen, um Ihre AetherFlow-Integrationen zu schuetzen.

<Columns cols={2}>
  <Card title="Token-Rotation" icon="refresh-cw">
    Rotieren Sie API-Token alle 90 Tage. Verwenden Sie verschiedene Token fuer verschiedene Umgebungen.
  </Card>
  <Card title="IP-Whitelisting" icon="lock">
    Beschraenken Sie den API-Zugriff fuer sensible Vorgaenge auf bestimmte IP-Adressen.
  </Card>
  <Card title="Ratenbegrenzung" icon="clock">
    Ueberwachen Sie Ihre API-Nutzung und implementieren Sie clientseitige Ratenbegrenzung.
  </Card>
  <Card title="Audit-Protokollierung" icon="file-text">
    Aktivieren Sie Audit-Protokolle, um die API-Nutzung zu verfolgen und Anomalien zu erkennen.
  </Card>
</Columns>

## Ratenlimits

AetherFlow setzt Ratenlimits durch, um eine faire Nutzung fuer alle Kunden zu gewaehrleisten.

| Plan | Anfragen pro Minute | Anfragen pro Stunde |
|------|---------------------|---------------------|
| Kostenlos | 60 | 1.000 |
| Pro | 300 | 10.000 |
| Enterprise | 1.000 | 50.000 |

<Callout kind="warning">
  Bei Ueberschreitung von Ratenlimits erhalten Sie HTTP-429-Antworten. Implementieren Sie exponentielles Backoff fuer Wiederholungsversuche.
</Callout>

## Datenverschluesselung

Alle Daten, die an AetherFlow-APIs uebertragen werden und von diesen empfangen werden, sind waehrend der Uebertragung mit TLS 1.3 verschluesselt. Sensible Daten werden im Ruhezustand mit AES-256-Verschluesselung gespeichert.

<ExpandableGroup>
  <Expandable title="Verschluesselungsdetails">
    - API-Kommunikation: TLS 1.3 mit Perfect Forward Secrecy
    - Datenbankverschluesselung: AES-256-GCM
    - Dateispeicherung: Serverseitige Verschluesselung mit kundenseitig verwalteten Schluesseln (Enterprise)
  </Expandable>
  <Expandable title="Compliance-Standards">
    AetherFlow haelt SOC-2-Typ-II- und DSGVO-Compliance fuer den Datenschutz aufrecht.
  </Expandable>
</ExpandableGroup>

Dieser umfassende Sicherheitsleitfaden stellt sicher, dass Sie AetherFlow sicher in Ihre Anwendungen integrieren koennen.
