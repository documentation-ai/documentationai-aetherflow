## Migration depuis Zapier

Guide étape par étape pour migrer de Zapier vers AetherFlow.

<Tabs>
  <Tab title="Exporter depuis Zapier" icon="download">
    1. Connectez-vous à votre compte Zapier

    2. Accédez à Paramètres > Personnel > Exporter vos données

    3. Demandez l'export des données (peut prendre 24 à 48 heures)

    4. Téléchargez le fichier JSON d'export lorsqu'il est prêt
  </Tab>

  <Tab title="Importer dans AetherFlow" icon="upload">
    1. Créez votre compte AetherFlow

    2. Accédez à Paramètres > Import/Export

    3. Sélectionnez « Importer depuis Zapier »

    4. Téléversez votre fichier JSON exporté

    5. Vérifiez et associez les intégrations

    6. Testez les flux de travail importés
  </Tab>

  <Tab title="Tâches post-migration" icon="check-circle">
    - Vérifier que toutes les intégrations sont connectées

    - Tester les flux de travail avec des données d'exemple

    - Mettre à jour les URL de webhook si nécessaire

    - Surveiller les journaux d'exécution pour détecter les problèmes

    - Optimiser les invites pour l'IA d'AetherFlow
  </Tab>
</Tabs>

## Migration depuis Microsoft Power Automate

Convertissez les Flows Microsoft en flux de travail AetherFlow.

<Expandable title="Processus de migration" default-open="false">
  1. **Exporter les Flows** : utilisez la fonctionnalité d'export de Power Automate pour télécharger les définitions des flows

  2. **Analyser la structure** : examinez la logique des flows, les conditions et les actions

  3. **Recréer dans AetherFlow** : utilisez le langage naturel pour décrire la logique des flows

  4. **Associer les intégrations** : connectez les intégrations AetherFlow équivalentes

  5. **Tester en profondeur** : validez les fonctionnalités avec des tests complets
</Expandable>

<Expandable title="Défis courants" default-open="false">
  - La logique conditionnelle complexe peut nécessiter une reformulation

  - Les connecteurs personnalisés doivent être recréés manuellement

  - Les flux d'approbation nécessitent une refonte pour le modèle d'AetherFlow

  - Certaines actions spécifiques à Power Automate n'ont pas d'équivalents directs
</Expandable>

## Plan de retour arrière

Préparez des plans de contingence en cas de problèmes lors de la migration.

<Expandable title="Stratégie de retour arrière" default-open="false">
  - **Exécution parallèle** : maintenez l'ancien système actif pendant la phase de test

  - **Bascule progressive** : migrez les flux de travail par phases plutôt que tout à la fois

  - **Sauvegarde des données** : conservez des sauvegardes de toutes les données critiques

  - **Retour rapide** : capacité à revenir à l'ancien système en quelques heures si nécessaire
</Expandable>
