## Create a workflow

Create a new workflow from your dashboard, choose what starts it, add the steps you want to run, and then save or publish it when you are ready.

<Callout kind="info">

Before you begin, sign in to your AetherFlow account and make sure you know the process you want to automate. If your workflow uses another app, connect that integration first or be ready to authorize it during setup.

</Callout>

## Step-by-step guide

<Steps>
  <Step title="Start a new workflow">
    From the AetherFlow dashboard, click **Create Workflow** or **New Workflow** to open the workflow builder.

    If you already have workflows, look for the create button in the top-right corner of the workflows page.

    You should now see a blank workflow draft or a setup screen where you can begin building.
  </Step>

  <Step title="Choose a trigger">
    Select the trigger that starts your workflow. The trigger might be an incoming email, a schedule, a form submission, or another event supported by your connected integrations.

    Pick the option that matches how the process should begin. If prompted, authorize the app or account the trigger depends on.

    When the trigger is set correctly, the builder shows it as the first step in your workflow.
  </Step>

  <Step title="Configure the trigger details">
    Add the information the trigger needs to run. Depending on the trigger, this might include a schedule, a mailbox, a form, or a specific event type.

    Review each setting before moving on. A clear trigger setup helps the workflow run at the right time and with the right data.

    After saving the trigger, you should be able to continue to the next step without validation errors.
  </Step>

  <Step title="Add your actions">
    Click **Add Step** to define what happens after the trigger runs. Actions are the tasks your workflow performs, such as sending a message, updating a record, creating a task, or passing information to another app.

    Add each action in the order it should run. If an action needs information from an earlier step, select the relevant data when configuring it.

    Your workflow should now show a sequence with one trigger followed by one or more action steps.
  </Step>

  <Step title="Review workflow settings">
    Check the workflow name, connected apps, and step order before saving. If the builder includes optional settings such as notifications or run preferences, review those now.

    This is also a good time to confirm that each step uses the correct account and expected inputs.

    When everything looks right, the workflow should appear complete and ready to save or publish.
  </Step>

  <Step title="Save or publish the workflow">
    Save the workflow if you want to finish it later, or publish it if you want it to start running. Use the available option in the builder to keep it as a draft or make it active.

    If the platform offers a test option, run a test before publishing to confirm the workflow behaves as expected.

    After publishing, the workflow status should show as active. If you saved it instead, it should remain available in your workflow list as a draft.
  </Step>
</Steps>

## Next steps

Once your workflow is created, test it with real or sample data so you can catch issues before relying on it. If you want to improve the workflow later, update the trigger, adjust the actions, or add more steps as your process changes.
