# Where can I find my API key?

API keys are usually available in the developer, API, or account settings for the service you want to connect. If you cannot find one, check that provider's security or developer documentation.

## Where API keys are usually managed

Most services place API keys in one of these areas:

- **Developer settings** for apps that support direct API access
- **API settings** for products with a dedicated developer section
- **Account or security settings** for services that treat keys as account credentials

Some providers hide the full key after you create it. In that case, you may need to reveal an existing key or generate a new one.

<Callout kind="info">

You may need admin or owner permissions to view, reveal, or rotate an API key. If the option is missing, ask your workspace administrator or check the provider's permission settings.

</Callout>

## Find or reveal your API key

<Steps>
  <Step title="Open the connected service settings">
    Sign in to the service you want to connect, then open its account, developer, or settings area.
  </Step>
  <Step title="Look for API or developer access">
    Search for sections labeled **API**, **Developers**, **Integrations**, **Security**, or **Access keys**.
  </Step>
  <Step title="Reveal or create a key">
    If a key already exists, look for an option to reveal or copy it. If no key exists, create a new one and copy it when it appears.
  </Step>
  <Step title="Store the key safely">
    Save the key in your password manager or secure internal vault before you leave the page. Many services only show the full key once.
  </Step>
</Steps>

## Keep your API key secure

Treat an API key like a password. Share it only with trusted systems and people who need it to complete the integration.

If you think a key was exposed, revoke or rotate it in the provider's settings, then update the integration with the new key.

## Where you use the key next

You usually paste the API key when setting up an integration, connecting a custom tool, or authorizing a service in your workflow. Once you have the key, return to the connection setup screen and enter it in the requested API key or authentication field.
