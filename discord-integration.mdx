## Overview

The Discord integration enables seamless communication between AetherFlow workflows and your Discord servers. Send automated notifications, post updates, create channels, and trigger workflows from Discord events. This integration supports both incoming and outgoing messages, webhooks, and real-time event handling.

## Setting Up Discord Integration

Navigate to the Integrations dashboard in AetherFlow and select Discord from the available options. You'll need Discord server admin permissions to complete the setup.

<Steps>
  <Step title="Create Discord Application" icon="settings">
    Go to [Discord Developer Portal](https://discord.com/developers/applications) and create a new application. Provide a name and accept the terms.
  </Step>
  <Step title="Configure Bot Settings" icon="key">
    Navigate to the "Bot" section and click "Add Bot". Enable the following permissions:
    - `Send Messages` - Post messages to channels
    - `Read Messages/View Channels` - Access channel content
    - `Embed Links` - Send rich embeds
    - `Attach Files` - Upload files
    - `Read Message History` - Access message history
    - `Manage Webhooks` - Create and manage webhooks
  </Step>
  <Step title="Get Bot Token" icon="download">
    Copy the bot token from the Bot section. Enable required Gateway Intents if using events.
  </Step>
  <Step title="Connect in AetherFlow" icon="link">
    Paste the token in AetherFlow's integration setup and authorize the connection.
    ```bash
    # Alternative: CLI setup
    aetherflow integrations connect discord --token your-bot-token
    ```
  </Step>
</Steps>

### Event Triggers

Set up workflows that respond to Discord events:

- **Message Events**: Trigger on specific keywords or patterns
- **Server Events**: React to new members or role changes
- **Reaction Events**: Process emoji reactions
- **Mention Events**: Respond when the bot is mentioned

Example trigger prompt:
```
When someone mentions @AetherFlow in Discord with "urgent", create a high-priority ticket and notify the on-call engineer.
```

## Webhook Integration

For real-time Discord events, configure webhooks:

<Steps>
  <Step title="Enable Gateway Intents" icon="webhook">
    In your Discord application settings, go to "Bot" and enable required Gateway Intents like "Server Members" and "Message Content".
  </Step>
  <Step title="Subscribe to Events" icon="bell">
    Configure which events to listen for: message creates, reactions, member updates.
  </Step>
  <Step title="Set Webhook URL" icon="link">
    Use your AetherFlow webhook URL: `https://api.aetherflow.com/webhooks/discord`
  </Step>
</Steps>

## Troubleshooting

### Common Issues

<ExpandableGroup>
  <Expandable title="Authentication Failed">
    Verify your bot token is correct and has the required permissions. Regenerate the token if needed.
  </Expandable>
  <Expandable title="Messages Not Posting">
    Check channel permissions and ensure the bot has access to the target channel.
  </Expandable>
  <Expandable title="Rate Limiting">
    Implement message queuing in your workflow. Discord allows 5 messages per 5 seconds per channel.
  </Expandable>
  <Expandable title="Webhook Events Not Received">
    Confirm Gateway Intents are enabled and the bot has proper permissions in your Discord server.
  </Expandable>
</ExpandableGroup>

## Security Considerations

<Callout kind="alert">
  Store Discord tokens securely and rotate them regularly. Use the minimum required permissions for your use case.
</Callout>

- **Token Management**: AetherFlow encrypts all integration tokens
- **Permission Limitation**: Only grant necessary bot permissions
- **Audit Logging**: All Discord interactions are logged for compliance
- **Access Control**: Integration access follows your workspace permissions

## Best Practices

1. **Channel Organization**: Use dedicated channels for different types of notifications
2. **Message Formatting**: Use Discord embeds for rich, interactive messages
3. **Error Handling**: Implement fallback channels for critical notifications
4. **Testing**: Always test integrations in a development server first
5. **Monitoring**: Set up alerts for integration failures
